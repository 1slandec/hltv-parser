# Анализ связи региона происхождения игроков Counter-Strike с рейтингом лучших команд мира HLTV

## Описание проекта

**Цель исследования**– выявить связь между регионом происхождения игроков в Counter-Strike и их положением в мировом рейтинге команд HLTV. 

**Объект исследования** – количество игроков из разных регионов (Европа, СНГ, Северная Америка, Южная Америка, Азия и др.) в различных диапазонах рейтинга (TOP 10, 50, 100, 200) за период с 2016 по 2026 год.

**Задачи:**
1. Сбор данных – парсинг страниц рейтинга команд HLTV.
2. Предобработка данных – определение региона для каждого игрока на основе страны.
3. Анализ данных – вычисление метрик (количество игроков, средняя/медианная позиция, взвешенный рейтинг, корреляции).
4. Визуализация – построение линейных, круговых, столбчатых диаграмм.

**Ожидаемые результаты:**
- Рейтинг регионов по количеству игроков в разных диапазонах рейтинга.
- Выявление периодов доминирования определённых регионов.
- Статистическая оценка связи между численностью игроков региона и их средним взвешенным рейтингом.

## Структура репозитория

- `parser.py` – скрипт для сбора данных с сайта HLTV.
- `analyse.py` – основной скрипт для анализа и визуализации данных.
- `requirements.txt` – зависимости Python.
- `data/` – папка для сохранения CSV-файлов и графиков (создаётся автоматически).
- `README.md` – описание проекта.

## Использование

### Сбор данных (парсинг)

Запустите скрипт `parser.py` для получения данных с HLTV:

```bash
python parser.py
```

Парсер последовательно загружает страницы рейтинга за январь (и март 2016) каждого года с 2016 по 2026. Между запросами установлена задержка 3 секунды. Результат сохраняется в файл `data/parsed-data.csv`.

Примечание: при первом запуске папка `data` будет создана автоматически.

### Анализ данных и визуализация

Запустите основной скрипт:

```bash
python analyse.py
```

После запуска отображается меню:

1. Полный текстовый анализ (для конкретного лимита рейтинга) – вычисляет корреляцию Пирсона между количеством игроков и взвешенным рейтингом, а также средний взвешенный рейтинг по регионам.
2. Построение и сохранение всех графиков (для конкретного лимита рейтинга) – пользователь вводит лимит (10, 50, 100 или 200) и получает линейный график долей, круговую диаграмму и столбчатую диаграмму взвешенного рейтинга.
3. Построение и сохранение всех графиков (для всех лимитов) – автоматически генерирует те же графики для TOP-10, TOP-50, TOP-100 и TOP-200.
4. Сравнение концентрации лидеров: Top-10 vs Top-200 – выводит матрицу коэффициентов (доля игроков в Top-10 к доле в Top-200) по годам.
5. Просмотр матриц данных – позволяет выбрать лимит и тип матрицы (общее количество, средняя позиция, медианная позиция, взвешенный рейтинг) и вывести её в консоль.
0. Выход из программы.

Все графики сохраняются в папку `data/` в формате PNG.

## Выходные данные

- `data/parsed-data.csv` – сырые данные с полями: `player`, `ranking_position`, `country`, `region`, `year`.
- `data/player_share_top_<limit>.png` – линейный график доли игроков по регионам.
- `data/avg_share_pie_chart_top_<limit>.png` – круговая диаграмма средней доли регионов.
- `data/avg_weighted_ranking_top_<limit>.png` – столбчатая диаграмма среднего взвешенного рейтинга.

## Источники данных

- HLTV.org – страницы рейтинга команд: `https://www.hltv.org/ranking/teams/`

## Примечания

- Парсинг выполняется с задержкой (time.sleep(3)), чтобы не перегружать сервер. При повторном запуске парсера данные будут собраны заново и перезапишут предыдущий CSV-файл.
- Скрипт `analyse.py` предполагает, что файл `data/parsed-data.csv` уже существует.
- Определение регионов основано на словаре стран (функция `get_region`). При появлении новых стран может потребоваться его расширение.
